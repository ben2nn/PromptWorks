# 提示词数据导入完整指南

## 📋 概述

本项目提供了完整的提示词数据导入解决方案，可以从 JSON 文件批量导入提示词到数据库。

## 🚀 快速开始（3步完成）

### 第1步：确保分类存在

```bash
python scripts/ensure_class.py
```

这将自动创建 `opennana` 分类（ID: 4）。

### 第2步：环境检查

```bash
python scripts/test_import.py prompts_20251025_192918.json
```

检查所有必要条件是否满足。

### 第3步：执行导入

```bash
python scripts/import_prompts.py prompts_20251025_192918.json
```

开始导入数据！

## 🔧 功能特性

### ✅ 自动化处理

- **标签管理**: 自动创建不存在的标签，分配颜色
- **图片下载**: 从 URL 自动下载图片到本地
- **缩略图生成**: 自动生成 200x200 缩略图
- **内容分离**: 智能识别并分离中英文内容
- **错误处理**: 单个失败不影响整体导入

### 📊 数据映射

```
JSON 数据                    数据库
├─ title                 → prompts.name
├─ content (英文)        → prompts_versions.content
├─ content (中文)        → prompts_versions.contentzh
├─ tags[]               → prompt_tags (多对多关联)
├─ image_url            → 下载后存储到 prompt_attachments
└─ category             → prompts.class_id (固定为 4)
```

### 内容分离规则

**方式1：使用分隔符**
```
English prompt content here

=== 提示词 ===

中文提示词内容
```

**方式2：自动判断**
- 中文字符 > 30% → 存入 `contentzh`
- 中文字符 ≤ 30% → 存入 `content`

## ⚠️ 注意事项

### 前置条件

1. ✅ 数据库已启动并可连接
2. ✅ 分类 ID 4 (opennana) 已存在
3. ✅ 有稳定的网络连接（下载图片）
4. ✅ 有足够的磁盘空间（存储图片）
5. ✅ 对 `uploads/` 目录有写权限

### 常见问题

**Q: 分类不存在怎么办？**
```bash
# 运行此脚本自动创建
python scripts/ensure_class.py
```

**Q: 图片下载失败？**
- 检查网络连接
- 图片下载失败不会阻止提示词创建
- 可以稍后手动上传图片

**Q: 导入速度慢？**
- 主要时间花在图片下载
- 可以先导入文本，后续批量处理图片

**Q: 如何避免重复导入？**
- 脚本不检查重复
- 重新运行会创建新记录
- 建议先清理旧数据或修改 JSON

## 🔍 数据验证

导入完成后，使用 SQL 验证：

```sql
-- 查看导入的提示词数量
SELECT COUNT(*) FROM prompts WHERE class_id = 4;

-- 查看创建的标签
SELECT name, color, COUNT(*) as usage_count
FROM prompt_tags pt
JOIN prompt_tag_links ptl ON pt.id = ptl.tag_id
GROUP BY pt.id, pt.name, pt.color
ORDER BY usage_count DESC;

-- 查看附件统计
SELECT 
    COUNT(*) as total_attachments,
    SUM(file_size) as total_size,
    AVG(file_size) as avg_size
FROM prompt_attachments 
WHERE prompt_id IN (SELECT id FROM prompts WHERE class_id = 4);

-- 查看最新导入的提示词
SELECT 
    p.id,
    p.name,
    p.author,
    COUNT(DISTINCT pa.id) as attachment_count,
    COUNT(DISTINCT ptl.tag_id) as tag_count,
    p.created_at
FROM prompts p
LEFT JOIN prompt_attachments pa ON p.id = pa.prompt_id
LEFT JOIN prompt_tag_links ptl ON p.id = ptl.prompt_id
WHERE p.class_id = 4
GROUP BY p.id, p.name, p.author, p.created_at
ORDER BY p.created_at DESC
LIMIT 10;
```

## 💡 高级用法

### 批量导入多个文件

```bash
# Windows PowerShell
Get-ChildItem *.json | ForEach-Object {
    python scripts/import_prompts.py $_.Name
}

# Windows CMD
for %f in (*.json) do python scripts/import_prompts.py %f
```

### 只导入部分数据

修改 JSON 文件，只保留需要的提示词：

```python
import json

# 读取原始文件
with open('prompts_20251025_192918.json', 'r', encoding='utf-8') as f:
    data = json.load(f)

# 只保留前 10 个
data['prompts'] = data['prompts'][:10]
data['metadata']['total_count'] = 10

# 保存新文件
with open('prompts_sample.json', 'w', encoding='utf-8') as f:
    json.dump(data, f, ensure_ascii=False, indent=2)
```
